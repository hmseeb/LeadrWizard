**Add rate-limited path prefixes** as a constant (before the middleware function):

```typescript
const RATE_LIMITED_PREFIXES = [
  "/api/webhooks/",
  "/api/widget/",
  "/api/signup/",
  "/api/cron/",
];

function isRateLimited(pathname: string): boolean {
  return RATE_LIMITED_PREFIXES.some((p) => pathname.startsWith(p));
}
```

These cover all 9 public endpoints from the research inventory. Authenticated endpoints (/api/billing/*, /api/org/*) are intentionally excluded since they're behind auth.

**Add at the START of the middleware function** (before the Supabase client creation):

```typescript
// --- Correlation ID ---
const correlationId = crypto.randomUUID();
request.headers.set("x-correlation-id", correlationId);

// --- Rate Limiting (public endpoints only) ---
if (isRateLimited(request.nextUrl.pathname)) {
  try {
    const ip =
      request.headers.get("x-forwarded-for")?.split(",")[0]?.trim() ??
      request.ip ??
      "unknown";
    const tier = getTierForPath(request.nextUrl.pathname);
    const limiter = getRateLimiter(tier);
    const { success, remaining, reset } = await limiter.limit(`${tier}:${ip}`);

    if (!success) {
      return NextResponse.json(
        { error: "Too many requests" },
        {
          status: 429,
          headers: {
            "Retry-After": String(Math.ceil((reset - Date.now()) / 1000)),
            "X-RateLimit-Remaining": "0",
            "X-RateLimit-Reset": String(Math.ceil(reset / 1000)),
          },
        }
      );
    }
  } catch {
    // If Upstash is unreachable (e.g. env vars missing in dev), fail open.
    // Don't block requests because of a rate limiter outage.
  }
}
```

Key implementation decisions:
- **Correlation ID is set on EVERY request**, not just rate-limited ones. This ensures all route handlers can use it for structured logging.
- **Rate limit key is `${tier}:${ip}`** so each endpoint group has independent limits. A client can make 100 webhook calls AND 60 widget calls per minute.
- **IP extraction** uses `x-forwarded-for` first (Vercel sets this), falls back to `request.ip`, then "unknown".
- **Fail open on error** (try/catch around rate limit check). If Upstash is down or env vars aren't set (local dev without Redis), requests pass through. This prevents the rate limiter from becoming a single point of failure.
- **429 response** includes `Retry-After` header (seconds until reset), `X-RateLimit-Remaining: 0`, and `X-RateLimit-Reset` (Unix timestamp). This follows RFC 6585.
