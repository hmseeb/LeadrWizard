<objective>
Install the stripe npm SDK and refactor stripe-adapter.ts to use the SDK client instead of raw fetch.

Purpose: SEC-01 requires `stripe.webhooks.constructEvent()` to verify webhook signatures. This function is only available via the stripe SDK — it cannot be replicated safely with raw crypto because it handles timestamp tolerance and constant-time comparison. The current stripe-adapter.ts uses raw `fetch` with a custom `stripeRequest()` helper.

Output: stripe SDK installed in packages/shared, stripe-adapter.ts exports a `constructEvent` function and uses the SDK's `stripe.checkout.sessions.create()` etc. replacing raw fetch calls.
</objective>
