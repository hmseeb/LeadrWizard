<objective>
Harden both webhook route handlers: add signature verification to the Stripe webhook, add idempotency deduplication to both, and remove the body.org_id fallback from the payment webhook.

Purpose: These are three of the five active exploits identified in Phase 1. The stripe route currently parses events without verifying they came from Stripe. The payment route can be exploited by passing body.org_id. Both routes are vulnerable to replay attacks because no idempotency check exists.

Output: stripe/route.ts verifies every request via constructEvent(), checks idempotency, and processes only new events. payment/route.ts checks idempotency on payment_ref and removes the body.org_id fallback.
</objective>

The POST handler must follow this exact sequence:

**Step 1 — Get raw body and signature** (already uses `request.text()` — keep this, do NOT switch to `request.json()`):
```typescript
const body = await request.text();
const sig = request.headers.get("stripe-signature");
if (!sig) {
  return NextResponse.json({ error: "Missing stripe-signature header" }, { status: 400 });
}
const webhookSecret = process.env.STRIPE_WEBHOOK_SECRET;
if (!webhookSecret) {
  throw new Error("STRIPE_WEBHOOK_SECRET environment variable is not set");
}
```

**Step 2 — Verify signature**:
```typescript
let event: Stripe.Event;
try {
  event = constructEvent(body, sig, webhookSecret);
} catch (err) {
  const message = err instanceof Error ? err.message : "Invalid signature";
  return NextResponse.json(
    { error: `Webhook signature verification failed: ${message}` },
    { status: 400 }
  );
}
```
Import `Stripe` type from `'stripe'` for the type annotation.

**Step 3 — Idempotency check** (before processing, prevents replay):
```typescript
const supabase = createServerClient();

const { data: existing } = await supabase
  .from("processed_webhook_events")
  .select("id")
  .eq("id", event.id)
  .maybeSingle();

if (existing) {
  return NextResponse.json({ received: true });  // Return 200, NOT 4xx/5xx
}

// Insert BEFORE processing to prevent race conditions
await supabase
  .from("processed_webhook_events")
  .upsert({ id: event.id, source: "stripe" }, { onConflict: "id", ignoreDuplicates: true });
```

**Step 4 — Process event**:
```typescript
await processStripeWebhook(supabase, event);
return NextResponse.json({ received: true });
```

IMPORTANT:
- Never return 5xx on a duplicate event — Stripe retries on 5xx, causing infinite loops
- `constructEvent` throws on invalid signature — catch it and return 400
- Keep the outer try/catch for unexpected errors (return 500 for genuinely unexpected failures)
- The `event` from `constructEvent` is a proper `Stripe.Event` — update `processStripeWebhook` call signature if needed (it accepts the event object directly)
